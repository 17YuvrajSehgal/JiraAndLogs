Modern Site Reliability Engineering teams have mature anomaly detectors, but the bottleneck has shifted from detection to diagnosis: identifying which past Jira ticket describes a recurring incident from current logs, traces, and metrics. No public dataset jointly contains rich telemetry and linked Jira issues, which blocks direct evaluation of memory-augmented retrieval for incident triage.

We construct such a dataset from a forked Google Online Boutique microservices demo with production-realistic OpenTelemetry instrumentation, twenty-seven scenario families across one hundred fault-injection runs, and a 347-ticket humanized Jira memory corpus, all assembled under a strict no-label-leakage discipline. We then introduce the Tiered Cascade Hybrid (\tch{}), a four-layer system composing seven Pareto-incomparable pipelines: a class-balanced logistic stacker for triage; Reciprocal Rank Fusion with a multi-retriever overlap-rerank for retrieval; an OR-combined novelty channel of agent verification, retrieval confidence, and a learned per-window classifier; and a final output composition.

On a 1{,}008-window in-distribution test split, \tch{} achieves Hit@5 of 0.912, MRR of 0.794, strict PR-AUC of 0.9998, and novel recall of 0.793 at preserved precision---a 388\% relative improvement over the locked baseline---at 16~$\mu$s per window after caching. The cascade transfers without retraining to the OpenTelemetry Demo, a structurally distinct polyglot application with Kafka-based asynchronous communication, and degrades gracefully under distractor noise and leave-one-family-out cross-validation. Three independent reranker designs each underperformed the cascade's overlap-rerank, isolating an operating regime in which multi-retriever consensus dominates single-model judgment.
